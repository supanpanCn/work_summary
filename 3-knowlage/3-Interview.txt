1-svg和canvas的区别
	二者均以2d为基本
	svg对分辨率要求不高、并且支持事件处理、适合做一些地图类项目
	canvas则依赖分辨率、不支持事件处理器、主战场在游戏领域
2-css3新增的伪类和伪元素
	before和after伪元素
	first-of-type last-of-type nth-of-type not empty last-child first-child
3-px、em和rem三种的区别
	px是绝对尺寸、em和rem的参照不同（em的参照基准是当前元素内的字体大小、rem则相对于根元素html而言）
4-@import和link标签引入外部样式表
	语法规则   <style>@import url('路径')</style>
		link标签
	加载顺序   link同步@import最后
	权重          link>@import
5-flex响应式布局
	排列方向(flex-direction）、是否换行(flex-wrap)、对齐方式(justify-content、align-items)

	类型：undefined、null、string、number、boolean、object、symbol
	返回值：undefined、object、function、string、number、boolean

	Number返回值不同（undefined的返回值为NaN，null的返回值为0）
	typeof的返回值
8-1和Number(1)的区别
	Number(1)将数字1转化为数字对象
9-严格模式的作用
	严格模式消除了js的一些不合理、不安全、和怪异行为，大大提高了开发人员的编程效率及减少了维护成本
	同时作为过渡性质为未来的js版本做铺垫
10-js如何定义一个类并实例化
	class Person{
		constructor(name,age){
			this.name = name
			this.age = age
		}
		say(){
			return 1
		}
	}
	
	function Person(name,age){
		this.name = name
		this.age = age
	}
	Person.prototype.say = function(){
		return 2
	}
	let person = new Person('sp',24)
11-闭包的优缺点
	优点：实现变量或者方法的私有化，一定程度上避免了全局变量的污染，她绕开了js的垃圾回收机制，延长了其生命周期
	缺点：内存泄漏、容易使代码变得复杂不利于项目开发
12-工厂模式和单例模式的区别
	两者都是作为对象车间而存在
	工厂创建的对象每一次都不一样，而单例则只在第一次创建随后直接return
13-通过在script标签中定义charset属性来规定编码的一致性
14-gif、jpg、png、webp
	gif：体积小、无损压缩 --- 颜色少
	jpg：渐进透明和动画不支持
	png：无损伤、支持透明、半透明、不透明 -- 不支持动画
	webp：体积小 -- 兼容
15-优化图片
	选用合理的图片格式
	优先css3
	预加载或懒加载
16-懒加载和预加载
	懒加载：在一定的契机触发加载，该契机即定义的可视区
	预加载：在页面渲染前加载图片，之后基于缓存渲染图片到页面
17-cookies，sessionStorage和localStorage的区别
	同：都是一种客户端的数据存储方式
	异：
		大小：cookie 4k 其他 5M
		有效期：cookie 窗口或浏览器关闭    sessionStorage：浏览器关闭    localStorage：永久
		交互：cookie 自动发送到服务器   其他 只在本地
